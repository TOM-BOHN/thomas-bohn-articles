# Contributing to Thomas Bohn Articles

Thank you for your interest in this repository!

## About Contributions

This repository contains original articles and frameworks by Thomas Bohn. The content reflects personal experience and professional insights from years of work in software engineering leadership and organizational transformation.

### Content Contributions

**Articles and frameworks are not accepting external contributions at this time.** The content represents the author's voice, experience, and perspective, which must remain consistent and authentic.

### What You Can Do

While we're not accepting content contributions, you can:

1. **Report Issues**
   - Typos or grammatical errors
   - Broken links or missing images
   - Technical issues with the repository
   - Formatting problems

2. **Suggest Improvements**
   - Clarifications needed in existing content
   - Additional examples that would help understanding
   - Questions about frameworks or their application

3. **Share Feedback**
   - How you've applied these frameworks
   - Results from using these approaches
   - Questions about specific scenarios

## How to Report Issues

If you find an issue, please:

1. Check if it's already been reported in [Issues](../../issues)
2. If not, create a new issue with:
   - Clear description of the problem
   - Location (article name, section)
   - Expected vs. actual behavior (for technical issues)
   - Screenshots if relevant

## Questions and Discussion

For questions about:
- **Framework application**: Open an issue with the "question" label
- **Technical implementation**: Open an issue with the "technical" label
- **General discussion**: Use GitHub Discussions (if enabled)

## Using These Frameworks

You're welcome to:
- Apply these frameworks in your own work
- Reference them in presentations or articles (with attribution)
- Adapt the methodology for your own projects
- Share your experiences using them

If you write about using these frameworks or adapt them for your context, we'd love to hear about it! Open an issue with the "showcase" label to share your work.

## Development System

The `.cursor/` directory contains the spec-driven writing system used to create these articles. You're welcome to:
- Study the approach
- Adapt it for your own projects
- Reference the methodology

If you adapt this system, please provide attribution and link back to this repository.

## Code of Conduct

This project follows a simple code of conduct:
- Be respectful and professional
- Provide constructive feedback
- Focus on the content and ideas, not personal attacks
- Assume good intent
