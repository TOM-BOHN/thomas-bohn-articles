# Visualization Strategy: Simple Whiteboard Style

**Strategy Name:** Simple Whiteboard Visual Style  
**Strategy Type:** Infographic Visualization Standard  
**Status:** Active  
**Last Updated:** 2026-01-27

---

## Overview

This visualization strategy defines a **simple whiteboard visual style** for creating infographics that make complex technical concepts approachable and easy to understand. The style emphasizes clarity, simplicity, and a collaborative whiteboard aesthetic.

**All infographics following this strategy MUST adhere to these specifications.**

---

## Visual Style: Whiteboard Drawing Aesthetic

**Overall Approach:** Simple, hand-drawn, sketch-like diagrams that make complex technical concepts approachable and easy to understand. The style should feel like a collaborative whiteboard session—professional yet informal, clear yet approachable.

---

## Color Palette

**Strictly Limited to Three Colors:**

- **Primary:** Black (#000000) for all outlines, text, and primary details
- **Background:** White (#FFFFFF) for the entire infographic background
- **Accent:** Light Blue (#ADD8E6 or similar) used sparingly for:
  - Icon fills within circular sections
  - Highlighting borders of key elements
  - Subtle emphasis on important visual elements

**No other colors** should be used—keep the palette strictly limited to these three.

---

## Visual Characteristics

### 1. Hand-Drawn / Sketch Aesthetic

- All lines and shapes should have a slightly irregular, informal quality
- Avoid perfectly straight lines or geometric precision
- Give the impression of being drawn by hand on a whiteboard
- **MUST include a subtle dotted line border around the entire image** to reinforce the sketch-like feel and create visual consistency

### 2. Simple, Conceptual Iconography

- Use minimalist line art icons consistently throughout
- Icons should directly represent concepts without excessive detail
- Examples: gears, clouds, bar charts, funnels, speedometers, compass roses, simple human figures
- Avoid photorealistic or highly detailed illustrations
- **Icon Usage Standard:** Use icons to represent key concepts, processes, or categories—one icon per major section or concept
- **Icon Placement:** Icons should be placed within circular sections or at the start of boxed sections
- **Icon Style:** Simple line art only, filled with Light Blue (#ADD8E6) accent color when inside circular sections

### 3. Typography

- **Main titles:** Bold, highly readable, hand-drawn style, approximately 1.5x larger than section headers
- **Section headers:** Bold, highly readable, hand-drawn style, approximately 1.2x larger than body text
- **Body text:** Clear and concise, maintaining readability, standard size
- Text should complement illustrative elements, not compete with them
- **Typography Hierarchy:** Maintain consistent size relationships—titles > headers > body text—across all infographics

### 4. Layout and Flow

- **Container Standardization:** Use **boxed sections** (rounded rectangles with slightly irregular edges) as the primary container type for consistency. Reserve circular sections only for special emphasis (e.g., central hubs, key concepts, North Star elements)
- **Layout Patterns:** Standardize on these layout patterns:
  - **Horizontal flow:** Left-to-right for processes, pipelines, and sequential flows
  - **Vertical stacking:** Top-to-bottom for problem-solution pairs, layered architectures, and hierarchical structures
  - **Three-section layouts:** Left-Center-Right for comparisons, value propositions, and stakeholder views
  - **Layered architectures:** Bottom-to-top for system layers, abstraction layers, and architectural stacks
- **Arrow Standardization:** Use **straight arrows with slightly irregular hand-drawn quality** (not perfectly straight) to show flow, relationships, and connections. Arrows should point clearly from source to destination. Use arrows consistently—one arrow per relationship or flow path
- **Spacing Standard:** Maintain consistent spacing between elements:
  - Minimum 1.5x element height between major sections
  - Minimum 1x element height between related sub-elements
  - Adequate white space (at least 20% of image area) to prevent clutter
- **Visual Hierarchy:** Use size and positioning to establish hierarchy:
  - Largest elements: Main concepts or central hubs
  - Medium elements: Supporting sections or key components
  - Smallest elements: Details, examples, or annotations
  - Maintain consistent size relationships across similar element types

### 5. Complexity Level

- **Keep it simple:** Focus on core concepts, not exhaustive detail
- Each infographic should communicate one main idea clearly
- Avoid overwhelming the viewer with too many elements
- **Element Limit:** Maximum 5-7 major visual elements per infographic
- Prioritize clarity over completeness

## Consistency Standards Summary

### Critical Consistency Requirements

1. **Border:** MUST include subtle dotted line border around entire image (not optional)
2. **Colors:** Black ONLY for all symbols (checkmarks, X marks, arrows, indicators)—NO green, red, or other colors
3. **Containers:** Boxed sections (rounded rectangles) as primary type; circular sections ONLY for special emphasis
4. **Layout Patterns:** Use one of four standard patterns (horizontal flow, vertical stacking, three-section, layered)
5. **Arrows:** Straight arrows with slightly irregular hand-drawn quality—one arrow per relationship
6. **Typography:** Maintain hierarchy: Titles 1.5x > Headers 1.2x > Body text
7. **Icons:** One icon per major section/concept, placed consistently
8. **Spacing:** Minimum 1.5x element height between major sections, 1x between related elements, 20% white space minimum
9. **Complexity:** Maximum 5-7 major visual elements per infographic

### Common Drift Areas to Avoid

- **Color violations:** Never mention "green checkmark" or "red X"—always use black
- **Layout inconsistency:** Don't mix layout patterns—choose one and stick to it
- **Container inconsistency:** Don't randomly mix circular and boxed sections—use boxed as primary
- **Arrow inconsistency:** Don't vary arrow styles—always use straight arrows with hand-drawn quality
