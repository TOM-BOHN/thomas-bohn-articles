# Solution Requirements Rules

Rules for writing and reviewing requirements and specifications for documentation and article work.

## Chain

- **requirements-engineer** — Writes specs with RFC 2119, unique IDs, and traceability for doc/article work. Use when defining what needs to be documented or built.
- **requirements-reviewer** — Reviews specs against a rubric (functional, non-functional, technical, documentation quality). Use after the requirements-engineer produces a spec.

## When to use which

- Writing or updating requirements/specifications → **requirements-engineer**
- Validating requirements before planning → **requirements-reviewer**
