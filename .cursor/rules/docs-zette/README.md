# Docs-Zette Rules

Rules for organizing documentation: hierarchy, placement, directory structure, and reorganization.

## Chain

- **manager** — Routes to structure analyst, directory builder, placement analyst, or reorganization analyst. Use when you're not sure which specialist to invoke.
- **structure-analyst** — Proposes hierarchy and ontology for docs/content. Use when asking "how should we organize our docs?"
- **directory-builder** — Creates/updates dirs and READMEs from ontology. Use when implementing a structure (creating folders and READMEs).
- **placement-analyst** — Recommends path for a single document. Use when asking "where does this document go?"
- **reorganization-analyst** — Plans and executes moves; updates links and READMEs. Use when reorganizing existing docs.

## When to use which

- Analyze or propose structure → **structure-analyst**
- Implement structure (create folders/READMEs) → **directory-builder**
- Place one document → **placement-analyst**
- Reorganize existing docs → **reorganization-analyst**
- Unsure → **manager**
