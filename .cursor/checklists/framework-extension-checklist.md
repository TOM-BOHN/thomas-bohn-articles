# Framework Extension Checklist

Use this checklist when adding to or expanding existing frameworks.

## Phase 1: Study the Original
- [ ] Read the original article multiple times
- [ ] Map out the framework structure
- [ ] Note terminology used for each element
- [ ] Understand the sequence and dependencies
- [ ] Identify the author's voice and style patterns

## Phase 2: Identify Gaps
- [ ] Compare framework to standards (origin story, definitions, sequence, gaps, measurement)
- [ ] Look for missing elements or incomplete sections
- [ ] Identify areas that need more examples
- [ ] Note where additional depth would add value
- [ ] Consider what questions readers might still have

## Phase 3: Maintain Consistency
- [ ] Use exact terminology from original article
- [ ] Match the formatting style (headings, lists, bold usage)
- [ ] Maintain the same voice and tone
- [ ] Follow the same structural patterns
- [ ] Use similar examples and scenarios

## Phase 4: Add Value
- [ ] New content provides actionable insights
- [ ] Examples are concrete and realistic
- [ ] Extensions address reader questions
- [ ] Additional depth enhances understanding
- [ ] No unnecessary redundancy

## Phase 5: Update Cross-References
- [ ] Check if other articles reference this framework
- [ ] Update table of contents if needed
- [ ] Add cross-links to related articles
- [ ] Update README with new content
- [ ] Verify all links work

## Ready to Publish
Once extension is complete, proceed to [Framework Quality Checklist](framework-quality-checklist.md)
