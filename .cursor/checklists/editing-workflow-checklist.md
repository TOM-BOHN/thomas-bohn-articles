# Editing Workflow Checklist

Use this checklist when editing or improving existing articles.

## Phase 1: Understand Intent
- [ ] Read the full article without making changes
- [ ] Identify the core message and framework
- [ ] Note what's working well
- [ ] Understand the target audience
- [ ] Identify the author's voice and style

## Phase 2: Identify Issues
- [ ] Create a list of issues found
- [ ] Categorize: clarity, flow, consistency, structure, technical
- [ ] Prioritize: critical vs. nice-to-have
- [ ] Note sections that need expansion or cutting
- [ ] Identify missing elements per framework standards

## Phase 3: Preserve Core
- [ ] Identify sentences/paragraphs that capture the author's voice
- [ ] Mark key insights that must be preserved
- [ ] Note unique examples or scenarios
- [ ] Preserve first-person narrative and experiential tone
- [ ] Keep the core structure unless fundamentally flawed

## Phase 4: Enhance Selectively
- [ ] Fix critical issues first (clarity, structure, technical)
- [ ] Improve examples by adding concrete details
- [ ] Strengthen transitions between sections
- [ ] Ensure terminology is consistent
- [ ] Add missing framework elements if needed
- [ ] Improve formatting and technical details

## Phase 5: Verify Consistency
- [ ] Voice is personal and experiential
- [ ] Tone matches other articles
- [ ] Terminology is consistent with repository
- [ ] Structure follows standard format
- [ ] Formatting matches other articles
- [ ] Quality standards met

## Ready to Publish
Once editing is complete, proceed to [Pre-Publication Checklist](pre-publication-checklist.md)
